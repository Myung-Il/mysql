백업의 개념
- 자원 예비품을 뜻함
- 예상 못한 문제를 해결하기 위해 DB 복제품을 보관하는 작업
- 복원은 백업해 둔 백업 파일을 사용해서 원래대로 만드는 것
미디어 오류 : DB가 저장된 매체의 고장으로 DB 사용이 불가할 때
사용자 오류 : 여러 사람이 쓰다가 데이터의 유실이나 변경 등
하드웨어 장애 : 자연재해나 물리적인 문제

백업의 종류
전체 백업
- 개체, 테이블, 데이터 전체를 백업함
- DB 복사본을 만드는 것이라고 보는게 좋음
- 전체를 복사하다보니 여러번 하면 중복이 많아져서 느림
- 최초 생성 때나 변경 사항이 있을 때 하는게 좋음
- 차등 백업 : 전체 백업 이후 변경된 사항이 있을 때 가능
- 증분 백업 : 트랜잭션 로그 파일을 저장하는 방법

MySQL의 백업 방법
[ Physical, Cold, Hot, Logical ]
- 물리적 백업 : DB 구동을 위한 모든 파일을 물리적으로 복사하는 방법
- 콜드 백업 : DB를 셧다운한 후 백업을 진행
- 핫 백업 : 운영 중인 DB의 파일을 복사하는 방법
- 논리적 백업 : 내용 별로 파일을 옮겨 백업하는 방법
